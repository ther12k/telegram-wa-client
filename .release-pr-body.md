## What ships

### Auth (Phase 1)

- `POST /api/auth/phone/start`, `POST /api/auth/code/submit`, `POST /api/auth/password/submit`, `POST /api/auth/qr/start`, `POST /api/auth/logout`
- `FakeTelegramAdapter` implementing the full state machine (`unauthenticated → requires_code → requires_password | authenticated`)
- Encrypted SQLite session storage + migration runner (`apps/server/test/migrations.test.ts`)
- Onboarding UI bound to real auth state — no more demo timeouts
- 7 adapter + 5 integration tests

### Dialogs (Phase 2)

- `GET /api/dialogs`
- Mutable in-memory dialog store (PATCH / DELETE mutations on chat id)
- `FixtureDialogProvider` with seeded chats (`alice`, `priya`, `mom`, `design-team`, `teletalk-updates`)

### Messaging (Phase 3)

- `GET /api/messages/:chatId` (history, cursor pagination)
- `POST /api/messages` (send; text OR mediaId via `.refine()`)
- `POST /api/messages/:chatId/read` (mark read)
- Optimistic send UI + ack reconciliation

### Real-time (Phase 4)

- Server-Sent Events stream at `GET /api/realtime/events`
- `RealtimeBus` pub/sub fan-out; auto-reconnect + replay on the frontend
- `message.new`, `message.update`, `dialog.update`, `presence`, `ping` event types

### Media (Phase 5)

- `POST /api/media` multipart upload → `mediaMetaSchema` (image / video / document / voice / sticker)
- `GET /api/media/:id` authenticated download
- Frontend bubbles: image preview, video player, document download, inline voice playback
- Upload spinner + optimistic-then-confirmed send

### V1 interactions (Phase 6)

- `GET /api/search?q=` — combined peer + message-text search
- `PATCH /api/dialogs/:chatId` — pin / mute / archive / unread mutations
- `DELETE /api/dialogs/:chatId`
- Frontend context menu (pin / archive / mute / unread / delete) wired to backend with optimistic UI + rollback
- `GlobalSearch` component now hits `/api/search` (debounced 300ms)

### Hardening (Phase 7)

- `securityHeaders` middleware — CSP, `X-Frame-Options: SAMEORIGIN`, `X-Content-Type-Options: nosniff`, HSTS
- `structuredLogger` middleware — JSON access logs (timestamp / requestId / method / url / status / durationMs / ip)
- `rateLimiter` middleware — sliding window 200 req / 60 s per IP on `/api/*`
- `x-request-id` propagated end-to-end

### Release (Phase 8)

- Multi-stage `Dockerfile` (oven/bun:1.3.14 build + runtime, non-root user, `HEALTHCHECK`, `APP_VERSION` / `APP_COMMIT` build args)
- `.dockerignore`
- `scripts/deploy.sh` — `--push`, `--ssh`, `--tag` modes
- `RUNBOOK.md` — architecture, deploy, health, ops, troubleshooting
